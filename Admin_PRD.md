## 1. 목적
판매자가 상품 등록, 라이브 일정·편성, 주문/결제 현황을 관리하고, 방송을 준비·운영·정산하는 데스크톱 백오피스를 제공한다.

## 2. 페르소나
- **P2 · 판매자/방송 운영자** — 데스크톱, 상품·방송·주문 총괄. 방송 전 준비 + 방송 중 실시간 모니터링 + 방송 후 성과 확인. **단일 관리자 계정 전제**(역할·권한 분리 비범위).

## 3. 기술 스택 (현행 유지)
React 18 · Vite 5 · TypeScript 5 · Tailwind CSS 3 · Zustand 5 · react-router-dom 6 · 인메모리 목데이터.

## 5. 기능 요구사항

### A1. 대시보드 ✅
- **FR-A1.1** 방송 요약(진행중 여부·제목·시청자·편성 상품수), 주문 수/매출/객단가, 베스트셀러 TOP5. (기존 로직 유지)
- **FR-A1.2** 입금대기·예외(만료/늦은입금) 건수 요약 + 목록 바로가기.

### A2. 상품관리 (CRUD · 옵션 · 옵션별 재고) 🔧
- **FR-A2.1** 상품 등록/수정/삭제. 기본정보(코드·상품명·카테고리·태그·정가·할인율·설명·상세·이미지).
- **FR-A2.2 (옵션)** 색상·사이즈 옵션 추가/삭제.
- **FR-A2.3 (옵션별 재고)** 색상×사이즈 **변형 매트릭스**를 생성하고 **변형별 재고 수량**과 품절임박 임계값을 관리. 옵션 없는 상품은 단일 변형 재고.
- **FR-A2.4** 재고 0 / 품절임박 상태를 목록에서 시각적으로 표시.
- **변경점**: 현행 목업의 `Product`에는 재고가 없고 재고가 라이브 편성 수량(`LiveItem.qty`)에 있었음 → **상품 변형 재고로 이전**.

### A3. 라이브 관리 (일정 · 편성 · 회차 복제) 🔧
- **FR-A3.1** 캘린더에서 세션 추가/선택/삭제, 일시·제목·외부 방송 URL 편집.
- **FR-A3.2** 선택 세션을 `live`로 전환(동시 1개: 기존 live는 upcoming 강등).
- **FR-A3.3 (편성=큐레이션)** 상품 피커로 노출 상품 선택, **노출 순서** 조정, **핀 상품** 지정, (선택) 방송 특가 설정. **재고 수량은 다루지 않음.**
- **FR-A3.4 (회차 복제)** 지난 세션 편성을 새 세션으로 복제.
- **변경점**: 편성에서 `qty(재고)` 개념 제거, 순서/핀/특가/복제 중심으로 재정의.

### A4. 주문관리 / 주문상세 ✅
- **FR-A4.1** 상태 필터별 주문 목록, 검색.
- **FR-A4.2 (상태전이)** 입금확인/만료/배송준비/송장입력·배송시작/배송완료/복구/환불. (기존 상태 머신 유지)
- **FR-A4.3** 주문상세: 구매자·배송지·메모·결제수단·라인 내역·금액·송장. 복구 시 **변형 재고 가용 검증**.

### A5. 예외관리 ✅
- **FR-A5.1** 자동만료/늦은입금 등 예외 주문 목록과 복구·만료 처리.

### A6. 설정 ✅
- **FR-A6.1** 배송비/무료배송 기준, 입금 계좌, 입금기한(시간), 결제수단 토글(카카오/토스/무통장), 배송메모 프리셋 관리. (User 결제 화면과 연동)

### A7. 관리자 로그인/계정 🆕
- **FR-A7.1** 관리자 로그인/로그아웃(목업 인증).
- **FR-A7.2** 계정 정보(아이디·비밀번호·연락처) 수정, 회원 탈퇴.
- **FR-A7.3** 비로그인 상태에서 백오피스 접근 시 로그인으로 리다이렉트.

### A8. 라이브 실시간 판매 콘솔 🆕
- **FR-A8.1** 현재 `live` 세션 기준 실시간 뷰: 편성 상품별 **판매 수량/잔여 재고 소진 현황**, 실시간 주문 목록, 누적 매출/주문수/객단가.
- **FR-A8.2** 품절/품절임박 상품 하이라이트(목데이터 기반 시뮬레이션 갱신 허용).
- **FR-A8.3** 편성 상품의 핀 지정/순서를 콘솔에서 즉시 조정(A3와 연동, User 홈에 반영).

### A9. 방송 회차별 성과 리포트 🆕
- **FR-A9.1** 세션(회차)별 매출·판매수량·주문수·객단가 비교.
- **FR-A9.2** 회차 내 상품별 성과(판매 TOP). (`SOLD_MAP` 확장, 목데이터)

### A10. 홈 큐레이션/기획전 관리 🆕
- **FR-A10.1** 방송이 없는 시간대에 구매자 홈(User FR-U1.2)에 노출할 상품 목록/순서 지정.
- **FR-A10.2** (선택) 기획전 배너 문구·다음 방송 예고 노출 설정.

## 6. 데이터 모델 (상위 PRD §7 기준)
- **재고는 상품 귀속 · 옵션(색상×사이즈) 변형 단위**. **라이브 세션은 재고 없이 편성(큐레이션)만** 보유.

```
Product { id, code, name, category, tag, price, discountRate, desc, details, images,
          options:{colors[],sizes[]}, variants: Variant[] }
Variant { color, size, stock, lowStockThreshold }
LiveSession { id, date, dow, start, end, title, status, videoUrl, pinnedId, items:CurationItem[] }
CurationItem { productId, order, salePriceOverride? }
Order { id, no, buyer{name,phone}, address, addrDetail, memo, depositor, method,
        items:OrderLine[], shippingFee, total, status, deadline, tracking, date }
OrderLine { productId, color, size, qty, unitPrice }
Settings { fee, freeOver, account, depositHrs, payments{...}, memos[] }
```
> 현행 목업 마이그레이션: `LiveItem.qty(재고)` → **Variant.stock**, 편성은 `CurationItem`으로.

## 7. 비즈니스 규칙
- **주문 상태 머신**: `wait→paid→ready→shipping→done` + 예외 `expired/late/refund/cancel`. 배송시작은 **송장번호 필수**. 복구는 **변형 재고 가용 검증** 통과 필요.
- **재고 차감/원복**: 주문 생성 시 차감, 취소/만료/환불 시 원복(정책 세부는 미해결).
- **방송중 판정**: `status === 'live'` 세션 **동시 최대 1개**.

## 8. 엑셀 Import / Export
- **FR-X.1** 상품 Import/Export(상품·변형·재고).
- **FR-X.2** 주문 Import/Export.
- **FR-X.3** 항목별 엑셀 템플릿 양식(컬럼 정의) 확정 필요(미해결).

## 9. 비범위
- 백엔드/실 API, 실 결제(PG), 배송사 연동, 관리자 역할·권한 분리/활동 로그, 별도 입금관리 전용 화면.

## 10. 미해결 이슈
> 아래는 미해결로 두며, 고객 요구에 따라 변동/구현한다.
1. 엑셀 템플릿 양식(상품/주문 Import·Export 컬럼 스키마).
2. 재고 원복 정책(취소/만료/환불별 세부 규칙).
3. 공용 도메인 타입 공유 방식(모노레포/공용 패키지 vs 앱별 복제).
4. 관리자 인증 실데이터 흐름 — 현 단계 목업, 향후 API 필요.
